## Сравнение LLM моделей: GLM-5 vs Kimi K2.5 vs MiniMax M2.5

### GLM-5 (Zhipu AI)

**Разработчик:** Zhipu AI (Китай)

**Особенности:**
- Флагманская модель Zhipu, ориентированная на **Agentic Engineering**
- Достигает **SOTA-уровня в кодинге** среди open-source моделей
- На бенчмарках SWE-bench Verified и Terminal Bench 2.0 — на уровне **Claude Opus 4.5**
- Полноценная экосистема: fine-tuning, multimodal (GLM-4.6V), агентные API
- Крупнейший независимый LLM-провайдер в Китае

**Сильные стороны:**
- Код и агенты — главные фишки
- Инструменты для бизнеса (перевод, генерация слайдов)
- Open-source компоненты

---

### Kimi K2.5 (Moonshot AI)

**Разработчик:** Moonshot AI (Китай)

**Особенности:**
- Фокус на **визуальном кодинге** и **Agent Swarm**
- Поддержка длинного контекста (исторически Kimi славилась этим)
- Инструменты: Deep Research, работа с документами, таблицами, слайдами
- Бета-режим Agent Swarm — множество агентов работают вместе

**Сильные стороны:**
- Visual coding — генерация кода из скриншотов/диаграмм
- Agent Swarm — координация нескольких агентов
- Работа с длинными документами

---

### MiniMax M2.5 (MiniMax)

**Разработчик:** MiniMax (Китай)

**Особенности:**
- **Мультимодальный стек:** текст, голос, видео, изображение, музыка
- Отдельные модели: MiniMaxMusic 2.5, MiniMaxHailuo 2.3 (видео), MiniMaxSpeech 2.6
- CodingPlan — дешёвый тариф для разработчиков
- MiniMaxAgent — универсальный помощник
- MiniMaxM2-her — ролевые модели для длинных диалогов

**Сильные стороны:**
- Самый широкий мультимодальный охват
- Видео и музыка — уникальные возможности
- Сверхнизкая задержка в Speech (для агентов)
- 130,000+ бизнес-клиентов
