### TypeScript Best Practices

**Ключевые практики:**
- Используй `strict mode` всегда — это база
- Избегай `any` → используй `unknown` или generics
- Prefer `interface` для shapes, `type` для unions
- Utility types: `Partial<T>`, `Pick<T, K>`, `Omit<T, K>` — твоя армия
- Branded types для runtime-безопасности (`type UserId = string & { readonly __brand: unique symbol }`)

**Ресурс:** https://www.typescriptlang.org/docs/handbook/declaration-files/do-s-and-don-ts.html
