Является ли код, который выполняется в основном потоке джаваскрипта макротаской?
assistant: Да, технически — **начальное выполнение скрипта** считается макротаской (task) в спецификации HTML.

**Как это работает:**

```
Event Loop
│
├─ 1. Выполнить одну macrotask
│     └─ Начальный код скрипта ← это macrotask
│
├─ 2. Выполнить все microtasks
│     └─ Promise.then/catch, queueMicrotask
│
├─ 3. UI Render (если нужно)
│
└─ Повторить →
```

**Пример:**

```javascript
console.log('1'); // ← синхронный код в начальной macrotask

setTimeout(() => console.log('2'), 0);  // → новая macrotask
Promise.resolve().then(() => console.log('3')); // → microtask

console.log('4'); // ← всё ещё начальная macrotask

// Порядок: 1 → 4 → 3 → 2
```

**Почему так:**
1. Весь синхронный код скрипта выполняется как одна **task** (macrotask)
2. После её завершения — event loop обрабатывает microtasks
3. Потом берёт следующую macrotask (setTimeout callback)

**Важно:** Не путай с синхронным кодом внутри callback'ов — callback от setTimeout это *отдельная* macrotask, а не часть начальной.
